# Brainstorming de Design - Wharith Bypass

<text>
### <idea>
**Design Movement**: Cyberpunk / Terminal Retro-Futurista
**Core Principles**:
1. **Imersão Total**: O usuário deve sentir que está acessando um sistema restrito e poderoso.
2. **Funcionalidade Crua**: A estética deve priorizar a sensação de utilidade e acesso direto ao código/sistema.
3. **Contraste Neon**: Fundo escuro profundo com acentos em ciano elétrico e roxo para destacar ações e status.
4. **Minimalismo Hacker**: Poucos elementos decorativos desnecessários, foco em texto, linhas de comando e feedback visual de sistema.

**Color Philosophy**:
- **Fundo**: `oklch(0.15 0.02 280)` (Deep Void) - Um preto quase absoluto, mas com um toque de azul para profundidade, evocando o "vazio" digital.
- **Primária**: `oklch(0.7 0.2 240)` (Cyan Glitch) - Para o logo, ícones e elementos de destaque, representando a energia do sistema.
- **Secundária**: `oklch(0.6 0.25 300)` (Purple Haze) - Para botões de ação e gradientes sutis, criando um contraste vibrante e misterioso.
- **Texto**: `oklch(0.9 0.05 240)` (Terminal White) - Branco levemente azulado para legibilidade em telas escuras, simulando monitores CRT modernos.

**Layout Paradigm**:
- **Terminal Centralizado**: O foco principal é a janela do terminal, que flutua no centro da tela, evocando uma interface de comando isolada.
- **Camadas de Profundidade**: O fundo não é chapado; possui elementos sutis (partículas, grid) que dão a sensação de um espaço tridimensional digital.
- **Rodapé Fixo**: Informações de autoria e links sociais ancorados na base, discretos mas acessíveis.

**Signature Elements**:
- **Scanlines Sutis**: Uma sobreposição muito leve de linhas horizontais para dar textura de monitor.
- **Efeito de "Glow"**: Textos e bordas importantes emitem um brilho suave, como luz neon.
- **Cursor Piscante**: O clássico cursor de terminal `_` ou `|` piscando para indicar prontidão.

**Interaction Philosophy**:
- **Feedback Imediato**: Botões reagem instantaneamente com mudanças de cor e brilho.
- **Digitação Automática**: Textos de sistema aparecem como se estivessem sendo digitados em tempo real.
- **Sons de Interface (Opcional/Visual)**: Animações que sugerem cliques mecânicos e processamento de dados.

**Animation**:
- **Entrada Glitch**: O logo e o terminal aparecem com um leve efeito de distorção digital na carga inicial.
- **Typing Effect**: Logs do terminal aparecem caractere por caractere.
- **Pulse**: O botão de "Injetar" pulsa suavemente para convidar à ação.

**Typography System**:
- **Display/Logo**: Uma fonte futurista e angular (ex: 'Orbitron' ou similar) para "WHARITH BYPASS".
- **Terminal/Código**: Uma fonte monoespaçada limpa e moderna (ex: 'JetBrains Mono', 'Fira Code' ou 'Share Tech Mono') para todo o conteúdo do terminal e logs.
</idea>
</text>

### <idea>
**Design Movement**: Glassmorphism Dark / Stealth Tech
**Core Principles**:
1. **Transparência e Desfoque**: Uso extensivo de fundos translúcidos com blur para criar uma interface moderna e sofisticada.
2. **Elegância Sombria**: Foco em tons de cinza escuro e preto, com luzes direcionais sutis.
3. **Hierarquia Visual Clara**: Uso de tamanho e peso de fonte para guiar o olho, em vez de cores vibrantes excessivas.
4. **Fluidez**: Transições suaves e elementos que parecem flutuar.

**Color Philosophy**:
- **Fundo**: `oklch(0.1 0 0)` (Pure Black) - Para máximo contraste e economia de energia em telas OLED.
- **Vidro**: `oklch(0.2 0 0 / 0.6)` - Camadas semitransparentes para o terminal e cartões.
- **Acento**: `oklch(0.75 0.15 250)` (Electric Blue) - Usado com parcimônia apenas para o logo e o botão principal.

**Layout Paradigm**:
- **Assimétrico Equilibrado**: O terminal pode estar ligeiramente deslocado, com elementos de fundo (como o logo grande e desfocado) criando equilíbrio.
- **Espaçamento Generoso**: Margens amplas para dar respiro e sofisticação.

**Signature Elements**:
- **Bordas Brilhantes**: Bordas finas com gradientes sutis para definir os limites dos elementos de vidro.
- **Sombras Coloridas**: Sombras difusas na cor do acento atrás dos elementos principais para criar "glow" ambiente.

**Interaction Philosophy**:
- **Hover Suave**: Elementos se elevam e brilham suavemente ao passar o mouse.
- **Micro-interações**: Ícones sociais reagem com preenchimento de cor ao hover.

**Animation**:
- **Fade-in em Cascata**: Elementos aparecem suavemente de baixo para cima, um após o outro.
- **Blur Dinâmico**: O fundo desfoca mais quando um modal ou ação importante é ativada.

**Typography System**:
- **Principal**: Uma fonte sans-serif geométrica e moderna (ex: 'Inter' ou 'Montserrat') para títulos e UI.
- **Código**: Fonte monoespaçada discreta para os logs do terminal.
</idea>

### <idea>
**Design Movement**: Brutalismo Digital / High-Tech Industrial
**Core Principles**:
1. **Estrutura Rígida**: Uso de linhas, grades e caixas visíveis para organizar a informação.
2. **Tipografia Grande**: Títulos enormes e impactantes.
3. **Cores de Alerta**: Uso de cores que remetem a avisos de sistema e perigo (vermelho, amarelo, ciano).
4. **Sem Sutilezas**: Bordas afiadas, sem sombras suaves, tudo muito direto e "na cara".

**Color Philosophy**:
- **Fundo**: `oklch(0.2 0.02 260)` (Dark Slate) - Um cinza azulado industrial.
- **Destaque**: `oklch(0.65 0.25 340)` (Neon Pink) e `oklch(0.8 0.2 200)` (Cyan) - Para criar um contraste vibrante e agressivo.

**Layout Paradigm**:
- **Grid Modular**: O site parece um painel de controle complexo, com o terminal sendo apenas um dos módulos.
- **Bordas Espessas**: Elementos delimitados por linhas grossas.

**Signature Elements**:
- **Marquee**: Textos rolando (scrolling text) com avisos ou status.
- **Ícones Pixelados**: Para reforçar a estética digital crua.

**Interaction Philosophy**:
- **Cliques Secos**: Botões mudam de estado instantaneamente, sem transição suave.
- **Hover Invertido**: Cores invertem (fundo vira texto, texto vira fundo) ao passar o mouse.

**Animation**:
- **Strobe**: Elementos piscam rapidamente em momentos críticos.
- **Slide Rápido**: Painéis deslizam para dentro da tela com velocidade.

**Typography System**:
- **Display**: Fontes monoespaçadas pesadas e expandidas.
- **Corpo**: Fontes sem serifa com alta legibilidade e aspecto técnico.
</idea>
